189. rotate_array

2019-11-13

Describtion:

Given an array, rotate the array to the right by k steps, where k is non-negative.

Example:

  • Example 1:

Input: [1,2,3,4,5,6,7] and k = 3 Output: [5,6,7,1,2,3,4] Explanation: rotate 1 steps to the right: [7,1,2,3,4,5,6] rotate 2 steps to the right: [6,7,1,2,3,4,5] rotate 3 steps to the right: [5,6,7,1,2,3,4]

  • Example 2:

Input: [-1,-100,3,99] and k = 2 Output: [3,99,-1,-100] Explanation: rotate 1 steps to the right: [99,-1,-100,3] rotate 2 steps to the right: [3,99,-1,-100]

  • Note:

Try to come up as many solutions as you can, there are at least 3 different ways to solve this problem. Could you do it in-place with O(1) extra space?

Thinking:

该题主要是通过给定的k值对现有数组进行位置移动, 且不使用额外空间实现解答 通过以上两点可以联想到数组的切片操作，且通过指定索引位置来获取一定范围区间的数字 通过计算移动位置 k % 数组长度可以得到移动的开始位置

    def rotate(self, nums, k):
    """
    :type nums: List[int]
    :type k: int
    :input [1,2,3,4,5,6,7] k=3
    :output [5,6,7,1,2,3,4]

    for idx in range(len(nums)):
      nums[idx+k] = nums[idx]
      # nums[3] = 0
      # nums[4] = 1

    解法1会报出以下错误： IndexError: list assignment index out of range 原因在于没有考虑超出数组索引范围的情况 (在数组位置向右移动 k 个位置的时候，偏后的数值其在数组中索引值是减小的) 这种解法只能对数组中前[k, len(nums)]个值进行赋值, 后续依次累加超出数组索引范围报错

  • solution1:( 基于解法1思路的升级版 ) > 如何先解决前 k 位的值，将后[k, n+k] 的值进行累加赋值给 nums?

    1. 由数组的切片方法可以得到： nums[:k] = nums[-k:]

    2. 通过遍历[k,len(nums)]范围的值，依次给 nums[k,len(nums)]的值进行赋值即可

      def rotate(self, nums, k):
"""
:type nums: List[int]
:type k: int
:input [1,2,3,4,5,6,7] k=3
:output [5,6,7,1,2,3,4]
"""
k = k % len(nums)
if k != 0:
  tmp = nums[-k:]
  for idx in range(k, len(nums)):
    nums[idx] = nums[idx-k]
  nums[:k] = tmp
else:
  return nums

  • solution2:(直接使用[-k:]+[:-k] 拼接完整数组)

    def rotate(self, nums, k):
    """
    :type nums: List[int]
    :type k: int
    :input [1,2,3,4,5,6,7] k=3
    :output [5,6,7,1,2,3,4]
    """
    target = k % len(nums)
    nums[:] = nums[-k:] + nums[:-k]

    Tips: 这道题里需要考虑到数组向右移动的位置数和数组长度之间的关系
    1. 移动值 < 数组长度 相当于数组从索引0的位置向右移动 k 个值
    2. 移动值 = 数组长度 相当于整个数组位置没有发生改变
    3. 移动值 > 数组长度 相当于数组从索引0的位置向右移动 k-len(nums) 个值
